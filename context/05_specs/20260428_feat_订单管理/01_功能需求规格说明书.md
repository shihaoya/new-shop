# 功能需求规格说明书

## 1. 概述

### 1.1 功能背景

订单管理是积分商城系统中连接店铺用户运营和普通用户兑换的核心功能模块。普通用户下单后产生订单，店铺用户对订单进行确认、发货、完成或关闭处理。系统区分虚拟商品和实物商品，实物商品需要物流配送（填写快递信息），虚拟商品则无需物流可直接完成。订单状态贯穿整个商品兑换生命周期，是积分扣除和库存扣减的关键触发点。

### 1.2 本功能业务目标

1. 为店铺用户提供完整的订单处理能力，支持订单的查看、确认、发货、完成、关闭
2. 为普通用户提供订单查看和申请关闭的能力，支持自主管理自己的订单
3. 区分虚拟商品和实物商品的订单流程，实物订单需要填写快递信息
4. 订单状态变更触发库存扣减和积分变动，确保数据一致性
5. 通过店铺审核机制保障订单操作的合规性

## 2. 角色与权限矩阵

> **说明**：本章节整合所有权限信息，是权限控制的**单一真实来源**。第4章各页面的权限要求均引用本章节，避免重复定义。

### 2.1 数据权限（行级访问控制）

> 定义不同角色针对**本功能数据**的行级别访问权限（即"谁能看到哪些数据行"）。

| 角色 | 数据范围 | 说明 |
|------|---------|------|
| 店铺用户 | 仅本店数据 | 只能查看和操作本店铺（shop_id）下的订单数据 |
| 普通用户 | 仅自己的数据 | 只能查看自己下的订单（通过 user_id 过滤） |

**数据范围说明：**
- **店铺用户**：通过 `shop_id = 当前用户所属店铺ID` 过滤，确保店铺间订单数据完全隔离
- **普通用户**：通过 `user_id = 当前用户ID` 过滤，确保用户只能查看自己下的订单

### 2.2 页面访问权限

> 定义不同角色对各页面的访问和操作权限。

| 页面名称 | 可访问角色 | 可操作角色 | 字段级权限（如有） |
|---------|----------|----------|------------------|
| 订单列表页（店铺侧） | 店铺用户 | 店铺用户 | 无特殊限制 |
| 订单详情页（店铺侧） | 店铺用户 | 店铺用户 | 无特殊限制（操作按钮根据订单状态显示） |
| 我的订单列表页 | 普通用户 | 普通用户 | 无特殊限制 |
| 我的订单详情页 | 普通用户 | 普通用户（仅申请关闭） | 仅在订单状态为「已下单」时显示「申请关闭」按钮 |

**权限说明：**
- **店铺用户**：可查看本店所有订单，可对订单进行确认/发货/完成/关闭操作
- **普通用户**：仅可查看自己下的订单，仅在「已下单」状态时可申请关闭订单
- **店铺状态约束**：店铺状态必须为「已通过」才能执行确认、发货、完成操作；查看操作无店铺状态限制

## 3. 页面与功能总览

### 3.1 页面清单

| 序号 | 页面名称 | 页面形式 | 职责说明 | 包含功能 |
|-----|---------|---------|---------|---------|
| 1 | 订单列表页（店铺侧） | 独立页面 | 展示本店所有订单，支持搜索、筛选、排序 | 查看订单列表、搜索订单、筛选订单、操作订单入口 |
| 2 | 订单详情页（店铺侧） | 独立页面 | 查看订单完整信息，进行订单操作 | 查看订单详情、确认订单、发货订单、完成订单、关闭订单 |
| 3 | 我的订单列表页 | 独立页面 | 展示当前用户下的订单，支持筛选、排序 | 查看我的订单列表、筛选订单 |
| 4 | 我的订单详情页 | 独立页面 | 查看订单完整信息，申请关闭订单 | 查看订单详情、申请关闭订单 |

### 3.2 页面跳转流程

```
【店铺用户侧】
订单列表页（店铺侧）
├── 点击「查看」→ 订单详情页（店铺侧） → 返回列表页
├── 点击「确认」→ 订单详情页（店铺侧） → 确认订单成功 → 刷新列表页
├── 点击「发货」→ 订单详情页（店铺侧） → 填写快递信息 → 发货成功 → 刷新列表页
├── 点击「完成」→ 订单详情页（店铺侧） → 完成订单成功 → 刷新列表页
└── 点击「关闭」→ 订单详情页（店铺侧） → 填写关闭理由 → 关闭订单成功 → 刷新列表页

【普通用户侧】
我的订单列表页
└── 点击「查看」→ 我的订单详情页 → 返回列表页

我的订单详情页
└── 点击「申请关闭」→ 填写关闭理由 → 关闭订单成功 → 刷新列表页
```

**流程说明：**
1. **店铺用户**：从侧边栏「订单管理」菜单进入订单列表页，可查看/搜索/筛选本店订单，点击进入详情页进行操作
2. **普通用户**：从侧边栏「我的订单」菜单进入订单列表页，可查看/筛选自己下的订单，在详情页可申请关闭「已下单」状态的订单

| 商品名称 | 文本 | 数据库查询 | 页面加载 | 无 | - | 兑换商品名称 |

**用户可操作功能：**

- **查看订单详情**
  - 触发按钮：「查看」按钮
  - 按钮级别：行级
  - 按钮位置：每行操作列
  - 触发方式：点击按钮
  - 权限要求：店铺用户
  - 加载状态：点击后显示loading，跳转至订单详情页
  - 功能说明：跳转至订单详情页，展示订单完整信息及可操作按钮

#### 4.1.4 业务规则

- **规则1：订单显示内容区分**
  - 规则描述：实物订单显示收货地址，虚拟订单不显示收货地址
  - 触发条件：列表页加载或详情页展示时
  - 约束说明：根据商品类别字段判断显示内容

#### 4.1.5 交互逻辑

- **列表展示方式**：表格视图
- **排序规则**：默认按下单时间倒序排列，支持时间升序/倒序切换
- **分页方式**：页码分页，每页显示条数可选（10/20/50）
- **空状态**：无订单时显示空状态提示「暂无订单」
- **加载状态**：首次加载和筛选时显示骨架屏或loading提示

#### 4.1.6 权限要求

> **参见第2.2节"页面访问权限"**，本页面的可访问角色、可操作角色、字段级权限均以第2章定义为准。

#### 4.2.3 功能与按钮

**用户可操作功能：**

- **确认订单**
  - 触发按钮：「确认订单」按钮
  - 按钮级别：页面级
  - 按钮位置：详情页操作区
  - 触发方式：点击按钮
  - 权限要求：店铺用户，且店铺状态为「已通过」
  - 显示条件：订单状态为「已下单」时显示
  - 加载状态：点击后显示loading，确认成功后刷新页面
  - 功能说明：直接将订单状态从「已下单」变更为「待发货」

- **发货订单**
  - 触发按钮：「发货」按钮
  - 按钮级别：页面级
  - 按钮位置：详情页操作区
  - 触发方式：点击按钮
  - 权限要求：店铺用户，且店铺状态为「已通过」
  - 显示条件：订单状态为「待发货」且商品类别为「实物」时显示
  - 加载状态：点击后显示loading，发货成功后刷新页面
  - 功能说明：填写快递类型和快递单号后，订单状态从「待发货」变更为「已发货」
  - 表单要求：快递类型（必填，1-10字）、快递单号（必填，1-30字）

- **完成订单**
  - 触发按钮：「完成订单」按钮
  - 按钮级别：页面级
  - 按钮位置：详情页操作区
  - 触发方式：点击按钮
  - 权限要求：店铺用户，且店铺状态为「已通过」
  - 显示条件：
    - 订单状态为「待发货」且商品类别为「虚拟」时显示（虚拟商品直接完成）
    - 订单状态为「已发货」时显示（实物商品需发货后才能完成）
  - 加载状态：点击后显示loading，完成成功后刷新页面
  - 功能说明：订单状态变更为「已完成」

- **关闭订单**
  - 触发按钮：「关闭订单」按钮
  - 按钮级别：页面级
  - 按钮位置：详情页操作区
  - 触发方式：点击按钮
  - 权限要求：店铺用户
  - 显示条件：订单状态为「已下单」「待发货」「已发货」时显示
  - 加载状态：点击后显示loading，关闭成功后刷新页面
  - 功能说明：填写关闭理由后，订单状态变更为「已关闭」
  - 表单要求：关闭理由（必填，1-50字）
  - 规则说明：关闭后订单终结，店铺用户可查看但不可再操作

#### 4.2.4 业务规则

- **规则1：虚拟商品订单流程**
  - 规则描述：虚拟商品订单无需填写快递信息，确认后可直接完成
  - 触发条件：订单状态为「待发货」且商品类别为「虚拟」时
  - 约束说明：只显示「完成订单」按钮，不显示「发货」按钮

- **规则2：实物商品订单流程**
  - 规则描述：实物商品订单必须填写快递类型和快递单号才能发货
  - 触发条件：订单状态为「待发货」且商品类别为「实物」时
  - 约束说明：必须先发货才能完成订单

- **规则3：关闭订单后订单终结**
  - 规则描述：订单关闭后状态为「已关闭」，店铺用户可查看详情但不可再进行任何操作
  - 触发条件：订单被关闭后
  - 约束说明：所有操作按钮隐藏，仅显示关闭信息和关闭时间

#### 4.2.5 交互逻辑

- **操作按钮显示逻辑**：根据订单状态和商品类别动态显示/隐藏操作按钮
- **表单校验**：发货和关闭操作需填写表单，校验不通过时显示错误提示
- **成功反馈**：操作成功后显示成功提示，页面自动刷新

#### 4.4.3 功能与按钮

**用户可操作功能：**

- **申请关闭订单**
  - 触发按钮：「申请关闭」按钮
  - 按钮级别：页面级
  - 按钮位置：详情页操作区
  - 触发方式：点击按钮
  - 权限要求：普通用户（仅自己订单）
  - 显示条件：订单状态为「已下单」时显示
  - 加载状态：点击后显示loading，关闭成功后刷新页面
  - 功能说明：填写关闭理由后，订单状态变更为「已关闭」
  - 表单要求：关闭理由（必填，1-50字）
  - 规则说明：关闭后订单终结，不可再进行任何操作

#### 4.4.4 业务规则

- **规则1：仅已下单状态可关闭**
  - 规则描述：普通用户只能在订单状态为「已下单」时申请关闭订单
  - 触发条件：查看订单详情时
  - 约束说明：订单状态为「待发货」「已发货」「已完成」「已关闭」时，「申请关闭」按钮不显示

- **规则2：关闭后订单终结**
  - 规则描述：订单关闭后状态为「已关闭」，普通用户可查看详情但不可再进行任何操作
  - 触发条件：订单被关闭后
  - 约束说明：「申请关闭」按钮隐藏

#### 4.4.5 交互逻辑

- **按钮显示逻辑**：仅在「已下单」状态时显示「申请关闭」按钮
- **表单校验**：申请关闭需填写理由，校验不通过时显示错误提示
- **成功反馈**：关闭成功后显示成功提示，页面自动刷新

### 5.1 订单状态自动触发

#### 5.1.1 库存和积分联动

| 触发时机 | 库存处理 | 积分处理 | 说明 |
|---------|---------|---------|------|
| 订单创建（已下单） | 预扣库存 | 预扣积分 | 库存和积分暂时锁定 |

- 订单创建时预扣库存和预扣积分，确保兑换资格
- 订单关闭时释放预扣资源，积分返还用户
- 订单完成时确认扣减，资源正式转换

## 6. 数据状态定义

### 6.1 数据状态

| 状态名称 | 状态说明 | 可转换到的状态 |
|---------|---------|--------------|
| 已下单 | 普通用户已成功兑换商品，积分已预扣 | 待发货、已关闭 |
| 待发货 | 店铺用户已确认订单，等待发货（虚拟商品可立即完成） | 已发货（实物）、已完成（虚拟）、已关闭 |
| 已发货 | 店铺用户已填写快递信息并发货 | 已完成、已关闭 |
| 已完成 | 订单已完成，积分和库存正式扣减 | 无（终态） |
| 已关闭 | 订单已关闭，积分和库存已释放 | 无（终态） |
