# 功能需求规格说明书

## 1. 概述

### 1.1 功能背景

商品管理是店铺用户对本店商品进行全生命周期管理的功能模块。店铺用户可以创建商品、编辑商品信息、上架下架商品、删除商品，以及查看商品列表和详情。商品是积分商城兑换系统的核心资源，连接店铺用户运营和普通用户兑换两个核心角色。

### 1.2 本功能业务目标

1. 为店铺用户提供完整的商品CRUD操作能力，支持商品的创建、编辑、查看、删除、上架、下架
2. 支持店铺用户对商品进行分类管理，便于商品组织和管理
3. 通过店铺审核机制确保商品上架的合规性，店铺必须通过审核才能操作商品
4. 区分虚拟商品和实物商品，虚拟商品无需物流，订单环节无需填写邮寄信息

## 2. 角色与权限矩阵

> **说明**：本章节整合所有权限信息，是权限控制的**单一真实来源**。第4章各页面的权限要求均引用本章节，避免重复定义。

### 2.1 数据权限（行级访问控制）

> 定义不同角色针对**本功能数据**的行级别访问权限（即"谁能看到哪些数据行"）。

| 角色 | 数据范围 | 说明 |
|------|---------|------|
| 店铺用户 | 仅本店数据 | 只能查看和操作本店铺（shop_id）创建的商品数据 |

**数据范围说明：**
- **仅本店数据**：通过 `shop_id = 当前用户所属店铺ID` 过滤，确保店铺间数据完全隔离

### 2.2 页面访问权限

> 定义不同角色对各页面的访问和操作权限。

| 页面名称 | 可访问角色 | 可操作角色 | 字段级权限（如有） |
|---------|----------|----------|------------------|
| 商品列表页 | 店铺用户 | 店铺用户 | 无特殊限制 |
| 商品创建/编辑弹框 | 店铺用户 | 店铺用户（需店铺状态为「已通过」） | 无特殊限制 |
| 商品详情弹框 | 店铺用户 | 店铺用户 | 仅查看，不可编辑 |

**权限说明：**
- **可访问角色**：能够进入该页面/弹窗的角色
- **可操作角色**：能够执行主要操作（新建、编辑、删除等）的角色
- **店铺状态约束**：店铺状态必须为「已通过」才能执行创建、编辑、上架、下架、删除操作；查看操作无店铺状态限制

## 3. 页面与功能总览

### 3.1 页面清单

| 序号 | 页面名称 | 页面形式 | 职责说明 | 包含功能 |
|-----|---------|---------|---------|---------|
| 1 | 商品列表页 | 独立页面 | 展示本店商品列表，支持搜索、筛选、分页 | 查看商品列表、搜索商品、筛选商品、新建商品入口、编辑商品入口、删除商品、上架/下架商品 |
| 2 | 商品创建/编辑弹框 | 弹框或抽屉 | 新建或编辑商品信息 | 创建商品、保存商品、编辑商品、保存修改 |
| 3 | 商品详情弹框 | 弹框或抽屉 | 查看商品完整信息 | 查看商品详情 |

### 3.2 页面跳转流程

```
商品列表页
├── 点击「新建」→ 商品创建弹框 → 保存成功 → 刷新列表页
├── 点击「编辑」→ 商品编辑弹框 → 保存成功 → 刷新列表页
└── 点击「查看」→ 商品详情弹框 → 关闭弹框 → 返回列表页
```

**流程说明：**
1. 用户从侧边栏菜单进入商品列表页
2. 在列表页点击「新建」按钮，打开商品创建弹框
3. 在列表页点击「编辑」按钮，打开商品编辑弹框（数据回填）
4. 在列表页点击「查看」按钮，打开商品详情弹框（仅查看）
5. 保存成功后自动刷新列表页数据

### 4.1 页面 1：商品列表页

#### 4.1.1 页面概述

**页面形式：**独立页面

**页面职责：**展示本店所有商品列表，支持搜索、筛选、分页查看。店铺用户可以在此页面管理商品（新建、编辑、上架、下架、删除）。

**用户进入场景：**店铺用户点击侧边栏「商品管理」菜单进入

#### 4.1.2 涉及字段

##### A. 查询字段（列表页专用）

| 字段名称 | 字段类型 | 数据来源 | 校验规则 | 默认值 | 业务含义 |
|---------|---------|---------|---------|--------|--------|
| 关键词搜索 | 文本 | 用户输入 | 支持模糊匹配 | 空 | 按商品名称搜索 |
| 商品分类 | 下拉选择 | 数据字典（商品分类管理） | 非必填 | 全部 | 按商品分类筛选 |
| 商品类别 | 下拉选择 | 系统预设 | 非必填 | 全部 | 按商品类别（虚拟/实物）筛选 |
| 状态 | 下拉选择 | 系统预设 | 非必填 | 全部 | 按上下架状态筛选 |

##### B. 显示字段

| 字段名称 | 字段类型 | 数据来源 | 获取时机 | 校验规则 | 默认值 | 业务含义 |
|---------|---------|---------|---------|---------|--------|--------|
| 商品名称 | 文本 | 数据库查询 | 页面加载 | 无 | - | 商品展示名称 |
| 商品主图 | 图片 | 数据库查询 | 页面加载 | 无 | - | 商品主图缩略图 |
| 积分价格 | 整数 | 数据库查询 | 页面加载 | 无 | - | 兑换所需积分 |
| 库存数量 | 整数 | 数据库查询 | 页面加载 | 无 | - | 当前库存，0表示无限（虚拟商品） |
| 商品分类 | 字典值 | 数据库查询 | 页面加载 | 无 | - | 所属分类名称 |
| 商品类别 | 字典值 | 数据库查询 | 页面加载 | 无 | - | 虚拟/实物 |
| 状态 | 标签 | 数据库查询 | 页面加载 | 无 | - | 已上架/已下架 |
| 创建时间 | 日期时间 | 数据库查询 | 页面加载 | 无 | - | 商品创建时间 |

#### 4.1.3 功能与按钮

**用户可操作功能：**

- **新建商品**
  - 触发按钮：「新建」按钮
  - 按钮级别：页面级
  - 按钮位置：列表页顶部工具栏
  - 触发方式：点击按钮
  - 权限要求：店铺用户，且店铺状态为「已通过」
  - 加载状态：点击后显示loading，弹出创建弹框
  - 功能说明：点击后打开商品创建弹框，用户填写商品信息后保存

- **编辑商品**
  - 触发按钮：「编辑」按钮
  - 按钮级别：行级
  - 按钮位置：每行操作列
  - 触发方式：点击按钮
  - 权限要求：店铺用户，且店铺状态为「已通过」
  - 加载状态：点击后显示loading，弹出编辑弹框
  - 功能说明：点击后打开商品编辑弹框，当前商品数据回填到表单

- **删除商品**
  - 触发按钮：「删除」按钮
  - 按钮级别：行级
  - 按钮位置：每行操作列
  - 触发方式：点击按钮
  - 权限要求：店铺用户，且店铺状态为「已通过」
  - 加载状态：点击后弹出确认对话框
  - 功能说明：删除前弹出确认对话框，确认后执行删除操作（逻辑删除）

- **上架商品**
  - 触发按钮：「上架」按钮
  - 按钮级别：行级
  - 按钮位置：每行操作列（商品下架时显示）
  - 触发方式：点击按钮
  - 权限要求：店铺用户，且店铺状态为「已通过」
  - 加载状态：点击后显示loading
  - 功能说明：将商品状态从「已下架」变更为「已上架」，商品可在商城展示

- **下架商品**
  - 触发按钮：「下架」按钮
  - 按钮级别：行级
  - 按钮位置：每行操作列（商品上架时显示）
  - 触发方式：点击按钮
  - 权限要求：店铺用户，且店铺状态为「已通过」
  - 加载状态：点击后显示loading
  - 功能说明：将商品状态从「已上架」变更为「已下架」，商品不在商城展示但保留数据

#### 4.1.4 业务规则

- **规则1：店铺状态约束**
  - 规则描述：店铺状态必须为「已通过」才能执行创建、编辑、上架、下架、删除操作
  - 触发条件：用户点击新建/编辑/删除/上架/下架按钮时
  - 约束说明：店铺状态为「审核中」「已拒绝」「已冻结」时，以上按钮不可点击或隐藏

- **规则2：删除确认**
  - 规则描述：删除商品前必须二次确认
  - 触发条件：点击删除按钮时
  - 约束说明：确认对话框说明删除影响，删除后商品数据不可恢复

#### 4.1.5 交互逻辑

- **列表展示方式**：支持卡片视图和表格视图切换，系统记住用户偏好
- **排序规则**：默认按创建时间倒序排列，支持点击表头排序
- **分页方式**：页码分页，每页显示条数可选（10/20/50）
- **空状态**：无商品时显示空状态提示和「新建商品」引导入口
- **加载状态**：首次加载和筛选时显示骨架屏或loading提示

#### 4.1.6 权限要求

> **参见第2.2节"页面访问权限"**，本页面的可访问角色、可操作角色、字段级权限均以第2章定义为准。

### 4.2 页面 2：商品创建/编辑弹框

#### 4.2.1 页面概述

**页面形式：**弹框对话框或抽屉

**页面职责：**新建商品或编辑已有商品信息。创建时为空表单，编辑时数据回填。

**跨页面功能归属说明：**新建商品入口在商品列表页（4.1.3），本页面仅包含完整的创建/编辑功能定义。

#### 4.2.2 涉及字段

| 字段名称 | 字段类型 | 数据来源 | 获取时机 | 校验规则 | 默认值 | 业务含义 |
|---------|---------|---------|---------|---------|--------|--------|
| 商品名称 | 文本 | 用户输入 | 实时输入 | 必填，2-100字符 | 空 | 商品展示名称 |
| 商品主图 | 文件上传 | 用户选择 | 实时输入 | 必填，1张图片，格式jpg/png/webp，≤5MB | 空 | 商品主图 |
| 商品详情图 | 文件上传 | 用户选择 | 实时输入 | 必填，1-5张图片，格式jpg/png/webp，每张≤5MB | 空 | 商品详情图（与主图分开上传） |
| 积分价格 | 整数 | 用户输入 | 实时输入 | 必填，正整数 | 空 | 兑换所需积分 |
| 库存数量 | 整数 | 用户输入 | 实时输入 | 必填，≥0的整数 | 空 | 当前库存，虚拟商品为0表示无限 |
| 商品分类 | 下拉选择 | 数据字典（商品分类管理） | 页面加载 | 必填 | 空 | 所属分类 |
| 商品类别 | 单选 | 用户选择 | 实时输入 | 必填，选项：虚拟/实物 | 空 | 商品属性，决定订单环节是否需填写邮寄信息 |
| 商品描述 | 富文本 | 用户输入 | 实时输入 | 非必填 | 空 | 商品详细描述 |
| 兑换限制 | 整数 | 用户输入 | 实时输入 | 默认0，≥0整数 | 0 | 单用户最大兑换次数，0表示无限制 |
| 状态 | 开关 | 用户选择 | 实时输入 | 必填 | 禁用（新建时默认下架） | 商品是否可兑换 |

#### 4.2.3 功能与按钮

**用户可操作功能：**

- **保存商品**
  - 触发按钮：「保存」按钮
  - 按钮级别：页面级
  - 按钮位置：弹框底部
  - 触发方式：点击按钮
  - 权限要求：店铺用户，且店铺状态为「已通过」
  - 加载状态：点击后显示loading，保存成功后关闭弹框并刷新列表
  - 功能说明：校验表单所有必填字段，校验通过后提交保存

- **取消**
  - 触发按钮：「取消」按钮
  - 按钮级别：页面级
  - 按钮位置：弹框底部
  - 触发方式：点击按钮
  - 权限要求：无
  - 加载状态：无
  - 功能说明：关闭弹框，不保存任何修改

#### 4.2.4 业务规则

- **规则1：新建时默认下架**
  - 规则描述：新建商品时，状态默认设置为「禁用」（已下架）
  - 触发条件：打开新建弹框时
  - 约束说明：需要手动上架后才能在商城展示

- **规则2：图片上传限制**
  - 规则描述：主图1张，详情图1-5张，均为jpg/png/webp格式，单张≤5MB
  - 触发条件：上传图片时
  - 约束说明：格式不符或大小超限时提示错误

#### 4.2.5 交互逻辑

- **实时校验**：字段失焦时进行校验，显示错误提示
- **图片预览**：上传后立即显示预览，支持删除和替换
- **弹框关闭**：关闭时清空表单内容，不保留任何修改

## 6. 数据状态定义

### 6.1 数据状态

| 状态名称 | 状态说明 | 可转换到的状态 |
|---------|---------|--------------|
| 已下架 | 商品创建后的默认状态，不在商城展示 | 已上架 |
| 已上架 | 商品已在商城展示，普通用户可兑换 | 已下架 |

### 6.2 状态转换规则

| 当前状态 | 可转换到的状态 | 转换条件 | 转换触发方式 |
|---------|------------|---------|-------------|
| 已下架 | 已上架 | 店铺状态为「已通过」 | 用户操作（上架按钮） |
| 已上架 | 已下架 | 店铺状态为「已通过」 | 用户操作（下架按钮） |

| 实物商品 | 需要物流配送的实体商品 | 订单环节需填写收货地址 |
| 商品类别 | 系统内置的商品分类，包括「虚拟」和「实物」两个选项 | 用于区分商品属性，决定订单流程 |
