# 功能需求规格说明书

## 1. 概述

### 1.1 功能背景

商品兑换是普通用户使用积分兑换本店商品的核心功能模块。普通用户通过商品列表浏览本店上架商品，选择商品进行兑换。兑换时根据商品类别（实物/虚拟）决定是否需要填写收货信息。兑换成功后将创建订单并扣除相应积分。

### 1.2 本功能业务目标

1. 为普通用户提供商品浏览和兑换能力，支持积分兑换商品
2. 区分虚拟商品和实物商品的兑换流程，虚拟商品无需填写收货信息
3. 支持地址簿功能，实物兑换时可选择已有地址或手动填写
4. 兑换时自动使用默认地址，减少用户操作步骤
5. 兑换成功后创建订单并扣除积分，确保积分与订单一致性

## 3. 页面与功能总览

### 3.1 页面清单

| 序号 | 页面名称 | 页面形式 | 职责说明 | 包含功能 |
|-----|---------|---------|---------|---------|
| 1 | 商品列表页（用户端） | 独立页面 | 展示本店已上架商品，支持搜索、筛选、分页 | 查看商品列表、搜索商品、筛选商品、进入商品详情、兑换商品 |
| 2 | 商品详情页（用户端） | 详情页 | 展示商品完整信息 | 查看商品详情、兑换商品 |
| 3 | 兑换表单弹框 | 弹框 | 填写收货信息（实物）或确认兑换（虚拟） | 填写收货信息、选择地址簿、确认兑换、取消 |
| 4 | 地址簿选择弹框 | 弹框 | 选择已有收货地址 | 查看地址列表、选择地址 |

### 3.2 页面跳转流程

```
商品列表页（用户端）
├── 点击「查看详情」→ 商品详情页（用户端）→ 点击「兑换」→ 兑换表单弹框
└── 点击「兑换」→ 兑换表单弹框

商品详情页（用户端）
└── 点击「兑换」→ 兑换表单弹框

兑换表单弹框（实物）
├── 点击「选择地址簿」→ 地址簿选择弹框 → 选择地址 → 自动填入表单
└── 点击「确认兑换」→ 扣除积分 → 创建订单 → 关闭弹框

兑换表单弹框（虚拟）
└── 点击「确认兑换」→ 扣除积分 → 创建订单 → 关闭弹框
```

**流程说明：**
1. 普通用户从侧边栏菜单进入商品列表页（用户端）
2. 在列表页点击商品可进入商品详情页
3. 在列表页或详情页点击「兑换」按钮，打开兑换表单弹框
4. 实物商品兑换时，显示收货信息表单，自动带入默认地址（如有）
5. 用户可点击「选择地址簿」从弹框中选择已有地址，自动填入表单
6. 用户也可手动填写/修改收货信息
7. 点击「确认兑换」后，系统扣除积分并创建订单

### 4.1 页面 1：商品列表页（用户端）

#### 4.1.1 页面概述

**页面形式：**独立页面

**页面职责：**展示本店已上架商品列表，支持搜索、筛选、分页查看。普通用户可以在此页面浏览商品并进入兑换流程。

**用户进入场景：**普通用户点击侧边栏「商品列表」菜单进入

#### 4.1.2 涉及字段

##### A. 查询字段（列表页专用）

| 字段名称 | 字段类型 | 数据来源 | 校验规则 | 默认值 | 业务含义 |

| 商品分类 | 下拉选择 | 数据字典（商品分类管理） | 非必填 | 全部 | 按商品分类筛选 |
| 商品类别 | 下拉选择 | 系统预设 | 非必填 | 全部 | 按商品类别（虚拟/实物）筛选 |

##### B. 显示字段

| 字段名称 | 字段类型 | 数据来源 | 获取时机 | 校验规则 | 默认值 | 业务含义 |

| 商品名称 | 文本 | 数据库查询 | 页面加载 | 无 | - | 商品展示名称 |
| 商品主图 | 图片 | 数据库查询 | 页面加载 | 无 | - | 商品主图缩略图 |
| 积分价格 | 整数 | 数据库查询 | 页面加载 | 无 | - | 兑换所需积分 |
| 库存数量 | 整数 | 数据库查询 | 页面加载 | 无 | - | 当前库存，0表示无限（虚拟商品） |
| 商品分类 | 字典值 | 数据库查询 | 页面加载 | 无 | - | 所属分类名称 |
| 商品类别 | 标签 | 数据库查询 | 页面加载 | 无 | - | 虚拟/实物 |

##### C. 按钮

| 按钮名称 | 按钮位置 | 按钮级别 | 可触发角色 | 功能说明 |
|---------|---------|---------|----------|---------|

| 积分不足提示 | 商品卡片/行 | 行级 | 普通用户（积分不足） | 显示提示，不可点击 |
| 库存不足提示 | 商品卡片/行 | 行级 | 普通用户（库存不足） | 显示提示，不可点击 |

#### 4.1.3 功能与按钮

**用户可操作功能：**

- **搜索商品**
  - 触发方式：输入关键词后回车或点击搜索图标
  - 按钮级别：字段级
  - 功能说明：按商品名称模糊搜索，支持实时搜索

- **筛选商品**
  - 触发方式：选择下拉选项
  - 按钮级别：字段级
  - 功能说明：按商品分类或商品类别筛选

- **查看商品详情**
  - 触发按钮：「查看详情」按钮
  - 按钮级别：行级
  - 按钮位置：每行操作列
  - 触发方式：点击按钮
  - 权限要求：普通用户
  - 功能说明：进入商品详情页，查看商品完整信息

- **兑换商品**
  - 触发按钮：「兑换」按钮
  - 按钮级别：行级
  - 按钮位置：每行操作列
  - 触发方式：点击按钮
  - 权限要求：普通用户，且积分足够且库存足够
  - 加载状态：点击后显示loading，弹出兑换表单弹框
  - 功能说明：打开兑换表单弹框，进入兑换流程
  - 禁用条件：积分不足或库存不足时按钮不可点击

#### 4.1.4 业务规则

- **规则1：仅显示已上架商品**
  - 规则描述：商品列表页只显示店铺用户已上架的商品
  - 触发条件：页面加载时
  - 约束说明：已下架商品不对普通用户可见

- **规则2：积分不足不可兑换**
  - 规则描述：用户积分不足时，兑换按钮不可点击
  - 触发条件：页面加载和积分变动时
  - 约束说明：实时检查用户积分与商品积分价格

- **规则3：库存不足不可兑换**
  - 规则描述：商品库存为0时（非虚拟商品无限库存场景），兑换按钮不可点击
  - 触发条件：页面加载时
  - 约束说明：虚拟商品库存为0表示无限库存，不受此限制

#### 4.1.5 交互逻辑

- **列表展示方式**：卡片视图为主，支持切换到表格视图
- **排序规则**：默认按创建时间倒序排列
- **分页方式**：页码分页，每页显示条数可选（10/20/50）
- **空状态**：无商品时显示空状态提示
- **加载状态**：首次加载和筛选时显示骨架屏或loading提示

#### 4.1.6 权限要求

> **参见第2.2节"页面访问权限"**，本页面的可访问角色、可操作角色、字段级权限均以第2章定义为准。

**页面形式：**弹框对话框

**页面职责：**实物商品填写收货信息，虚拟商品直接确认兑换。完成兑换后扣除积分并创建订单。

**跨页面功能归属说明：**兑换入口在商品列表页和商品详情页（4.1.3、4.2.3），本页面包含完整的兑换功能定义。

- **规则1：默认地址自动带入**
  - 规则描述：实物商品兑换时，如用户有默认地址，则自动填充到收货信息表单
  - 触发条件：打开兑换表单弹框时（实物商品）
  - 约束说明：如无默认地址，则表单为空

- **规则2：积分扣除时机**
  - 规则描述：积分在确认兑换时立即扣除，不支持预占或锁定
  - 触发条件：点击确认兑换按钮，校验通过后
  - 约束说明：扣除积分前再次检查积分是否足够

- **规则3：库存扣减时机**
  - 规则描述：库存在下单时立即扣减
  - 触发条件：点击确认兑换按钮，校验通过后
  - 约束说明：虚拟商品无库存限制

- **规则4：单用户兑换限制**
  - 规则描述：如商品设置了兑换限制（非0），则检查用户已兑换次数
  - 触发条件：点击确认兑换按钮时
  - 约束说明：已兑换次数达到上限时提示不可兑换

#### 4.3.5 交互逻辑

- **表单差异**：
  - 实物商品：显示收货人姓名、手机号、收货地址三个输入框，以及「选择地址簿」按钮
  - 虚拟商品：不显示邮寄信息表单，直接显示订单确认信息
- **实时校验**：字段失焦时进行校验，显示错误提示
- **弹框关闭**：关闭时清空表单内容
- **兑换成功**：显示成功提示，关闭弹框

#### 4.3.6 权限要求

> **参见第2.2节"页面访问权限"**，本页面的可访问角色、可操作角色、字段级权限均以第2章定义为准。

「无额外业务规则」

## 6. 数据状态定义

### 6.1 数据状态

| 状态名称 | 状态说明 | 可转换到的状态 |
|---------|---------|--------------|
| 已下单 | 普通用户兑换成功，订单已创建，等待店铺确认 | 已确认、已关闭 |
| 已确认 | 店铺用户已确认订单 | 已发货、已关闭 |
| 已发货 | 店铺用户已发货（实物） | 已完成 |
| 已完成 | 订单已完成 | 无 |
| 已关闭 | 订单已关闭（用户或店铺申请关闭） | 无 |

### 6.2 状态转换规则

| 当前状态 | 可转换到的状态 | 转换条件 | 转换触发方式 |
|---------|------------|---------|-------------|
| 已下单 | 已确认 | 店铺用户确认订单 | 用户操作 |
| 已下单 | 已关闭 | 用户或店铺申请关闭 | 用户操作 |
| 已确认 | 已发货 | 店铺用户填写物流信息 | 用户操作 |
| 已确认 | 已关闭 | 店铺用户申请关闭 | 用户操作 |
| 已发货 | 已完成 | 店铺用户确认收货或系统自动确认 | 用户操作/系统自动 |
