# 功能需求规格说明书

## 1. 概述

### 1.1 功能背景

为店铺用户提供本店普通用户管理能力，包括用户查看、创建、编辑、密码重置、积分调整等完整用户生命周期管理。店铺用户只能管理本店普通用户数据，无法访问其他店铺用户信息。

### 1.2 本功能业务目标

1. 实现普通用户的增删改查基本操作，支持批量导入
2. 支持行内快速调整积分，保留完整积分变动记录
3. 提供密码重置功能，系统随机生成8位密码并通过弹框展示
4. 导入功能采用全成功或全失败策略，确保数据一致性

#### 4.1.4 业务规则

- **规则1：数据隔离**
  - 规则描述：列表页仅显示当前店铺的普通用户数据
  - 触发条件：页面加载时自动过滤
  - 约束说明：通过 shop_id 过滤实现

- **规则2：用户名唯一性**
  - 规则描述：用户名（不算逻辑删除的用户）全局唯一
  - 触发条件：新增用户、导入用户时校验

- **规则3：状态显示**
  - 规则描述：列表页显示用户当前状态
  - 触发条件：列表加载
  - 约束说明：状态值包括待审核/已通过/已拒绝/已冻结

#### 4.1.5 交互逻辑

- 列表默认按注册时间倒序排列（最新注册的用户在前）

### 4.7 导入用户弹窗

#### 4.7.1 页面概述

**页面形式：**弹窗对话框

**页面职责：**通过Excel批量导入普通用户，全成功或全失败。

#### 4.7.2 涉及字段

| 字段名称 | 字段类型 | 数据来源 | 获取时机 | 校验规则 | 默认值 | 业务含义 |
|---------|---------|---------|---------|---------|--------|--------|
| 文件上传 | 文件 | 用户选择 | 用户选择文件 | 必填，.xlsx格式 | 无 | Excel文件 |

**导入模板字段：**

| 字段名称 | 字段类型 | 是否必填 | 说明 |
|---------|---------|---------|------|
| 用户名 | 文本 | 是 | 4-20字符，字母数字 |
| 昵称 | 文本 | 否 | 2-20字符 |
| 初始积分 | 数字 | 否 | 默认0，>=0整数 |

#### 4.7.3 功能与按钮

**用户可操作功能：**

- **上传文件**
  - 触发按钮：选择文件按钮
  - 按钮级别：页面级
  - 按钮位置：弹窗中央
  - 触发方式：点击按钮选择文件
  - 权限要求：店铺用户
  - 加载状态：上传中显示loading
  - 功能说明：选择Excel文件

- **确认导入**
  - 触发按钮：弹窗底部"确认"按钮
  - 按钮级别：页面级
  - 按钮位置：弹窗底部
  - 触发方式：点击按钮
  - 权限要求：店铺用户
  - 加载状态：导入过程中显示loading
  - 功能说明：执行导入，全成功全失败

- **取消关闭**
  - 触发按钮：弹窗底部"取消"按钮
  - 按钮级别：页面级
  - 按钮位置：弹窗底部
  - 触发方式：点击按钮
  - 权限要求：店铺用户
  - 加载状态：无
  - 功能说明：关闭弹窗

**后台自动流程（如有）：**

无

#### 4.7.4 业务规则

- **规则1：全成功全失败**
  - 规则描述：导入要么全部成功，要么全部失败，不允许部分成功
  - 触发条件：导入Excel时
  - 约束说明：任何一行数据校验失败，全部回滚

- **规则2：文件格式校验**
  - 规则描述：仅支持.xlsx格式文件
  - 触发条件：选择文件时
  - 约束说明：格式错误提示"请上传.xlsx格式文件"

- **规则3：模板字段校验**
  - 规则描述：每行数据校验用户名（必填，4-20字符，字母数字）、昵称（选填，2-20字符）、初始积分（选填，默认0，>=0整数）
  - 触发条件：导入时
  - 约束说明：校验失败显示行号和具体错误

- **规则4：用户名唯一性**
  - 规则描述：用户名全局唯一（不算逻辑删除的用户）
  - 触发条件：导入时
  - 约束说明：重复时报错

- **规则5：密码生成**
  - 规则描述：每个导入成功的用户系统随机生成8位密码
  - 触发条件：导入成功时
  - 约束说明：成功弹框显示所有用户的用户名和新密码

#### 4.7.5 交互逻辑

- 选择文件后自动开始导入，无需点击导入按钮
- 导入过程中显示进度
- 成功时弹框显示所有用户的用户名和密码表格（可复制、可导出）
- 失败时弹框显示所有错误行及具体原因，可滚动查看
- 错误信息格式："第N行：[字段名] - [错误原因]"

## 6. 数据状态定义

### 6.1 数据状态

| 状态名称 | 状态说明 | 可转换到的状态 |
|---------|---------|--------------|
| 待审核 | 新创建的用户，待店铺用户审核 | 已通过、已拒绝 |
| 已通过 | 审核通过的用户，可正常登录和使用 | 已冻结 |
| 已拒绝 | 审核被拒绝的用户 | 无（终态） |
| 已冻结 | 被冻结的用户，无法登录 | 已通过（解冻） |

### 6.2 状态转换规则

| 当前状态 | 可转换到的状态 | 转换条件 | 转换触发方式 |
|---------|------------|---------|-------------|
| 待审核 | 已通过 | 店铺用户审核通过 | 用户操作 |
| 待审核 | 已拒绝 | 店铺用户审核拒绝 | 用户操作 |
| 已通过 | 已冻结 | 店铺用户执行冻结操作 | 用户操作 |
| 已冻结 | 已通过 | 店铺用户执行解冻操作 | 用户操作 |
