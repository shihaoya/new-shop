# 功能需求规格说明书

## 1. 概述

### 1.1 功能背景

本系统为多店铺积分商城系统，需要在业务功能开发前完成底层基础设施建设，确保：
- 多店铺数据隔离（shop_id机制）
- 系统枚举值可配置管理（数据字典）
- 用户身份认证和会话管理（登录/登出）
- 邀请码和消息通知等公共服务

### 1.2 本功能业务目标

1. **数据隔离**：通过shop_id租户隔离确保各店铺数据完全独立，所有数据操作自动带上租户上下文
2. **数据字典管理**：为系统提供统一的枚举值管理能力，支持管理员动态配置业务枚举值
3. **用户认证**：提供安全的用户登录/登出机制，包含Token生成和验证
4. **邀请码服务**：提供邀请码的生成、验证和状态管理能力
5. **消息通知服务**：提供站内消息和短信通知能力，支持审核结果、订单超时等通知场景

## 2. 角色与权限矩阵

### 2.1 数据权限（行级访问控制）

> 基础设施为系统底层能力，被动受所有业务模块依赖。数据隔离通过shop_id自动过滤，无需用户层面配置。

| 角色 | 数据范围 | 说明 |
|------|---------|------|
| 管理员（平台层） | 全部数据（跨店铺） | 管理员可查看和管理所有店铺数据，通过shop_id区分 |
| 店铺用户（店铺层） | 本店数据 | 通过shop_id过滤，只能操作本店数据 |
| 普通用户（用户层） | 本店数据 | 通过shop_id过滤，只能操作本店数据 |

**数据范围说明：**
- **shop_id自动过滤**：所有数据查询自动附加shop_id条件，由基础设施层拦截处理
- **租户上下文**：用户登录时注入shop_id，后续所有操作携带该上下文

### 2.2 页面访问权限

> 基础设施中需要用户界面的功能：登录页、数据字典管理页。

| 页面名称 | 可访问角色 | 可操作角色 | 字段级权限（如有） |
|---------|----------|----------|------------------|
| 登录页 | 所有未登录用户 | 所有未登录用户 | 无 |
| 数据字典管理页 | 管理员 | 管理员 | 无 |

### 4.1 登录页

#### 4.1.1 页面概述

**页面形式：**独立页面

**页面职责：**提供用户身份认证入口，验证账号密码后生成Token。系统初始化时内置第一个管理员账号。

#### 4.1.2 涉及字段

##### A. 查询字段（列表页专用）

> 本页面为登录表单，无查询字段

##### B. 显示字段

| 字段名称 | 字段类型 | 数据来源 | 获取时机 | 校验规则 | 默认值 | 业务含义 |
|---------|---------|---------|---------|---------|--------|--------|
| 账号 | 文本 | 用户输入 | 实时输入 | 必填，4-50字符 | 空 | 用户登录账号 |
| 密码 | 密码 | 用户输入 | 实时输入 | 必填，6-20字符 | 空 | 用户登录密码 |
| 验证码 | 文本 | 用户输入 | 实时输入 | 必填，4位数字 | 空 | 图片验证码（可选） |

#### 4.1.3 功能与按钮

**用户可操作功能：**

- **登录**
  - 触发按钮：登录按钮
  - 按钮级别：页面级
  - 按钮位置：表单底部
  - 触发方式：点击按钮
  - 权限要求：所有未登录用户
  - 加载状态：点击后显示loading，禁止重复提交
  - 功能说明：验证账号密码，登录成功返回Token，登录失败提示错误原因

- **登出**
  - 触发按钮：登出按钮（位于各业务页面顶部导航栏）
  - 按钮级别：页面级
  - 按钮位置：顶部导航栏右侧
  - 触发方式：点击按钮
  - 权限要求：已登录用户
  - 加载状态：点击后清除Token，跳转登录页
  - 功能说明：清除本地Token，销毁会话，重定向到登录页

**后台自动流程（如有）：**

- **Token自动刷新**
  - 触发时机：Token即将过期前5分钟
  - 说明：前端自动调用刷新接口，延长Token有效期

#### 4.1.4 业务规则

- **规则1：账号密码校验**
  - 规则描述：账号和密码必须同时正确才能登录成功
  - 触发条件：用户点击登录按钮
  - 约束说明：密码错误时提示"账号或密码错误"，不区分具体是账号错还是密码错

- **规则2：登录失败限制**
  - 规则描述：同一账号连续5次登录失败后，锁定30分钟
  - 触发条件：登录失败次数达到阈值
  - 约束说明：锁定期间该账号不允许登录

- **规则3：Token有效期**
  - 规则描述：登录成功后返回的Token有效期为24小时
  - 触发条件：登录成功
  - 约束说明：过期后需重新登录，支持Token刷新延长有效期

#### 4.1.5 交互逻辑

- 账号输入框：支持回车键快捷登录
- 密码输入框：支持回车键快捷登录，回车后自动触发登录
- 登录失败：表单顶部显示红色错误提示，账号密码清空，聚焦到账号输入框
- 登录成功：跳转到系统首页或登录前页面

> 参见第2.2节

- **规则2：分类保护**
  - 规则描述：系统内置分类（如订单状态、商品分类等）不允许删除
  - 触发条件：删除字典分类时
  - 约束说明：内置分类的删除按钮置灰或隐藏

- **规则3：字典项保护**
  - 规则描述：已被其他数据引用的字典项不允许删除
  - 触发条件：删除字典项时
  - 约束说明：删除前检查引用关系，存在引用时提示"该字典项已被使用，无法删除"

- **规则4：禁用时不可用**
  - 规则描述：状态为"禁用"的字典项在业务选择中不可见
  - 触发条件：业务模块查询可用的字典项时
  - 约束说明：查询时自动过滤禁用状态

## 5. 非页面功能详细说明

### 5.1 数据隔离基础设施

#### 5.1.1 功能概述

**触发方式：**自动拦截
**触发时机：**所有数据操作自动触发
**功能职责：**确保多店铺数据完全隔离，通过shop_id过滤实现租户隔离

#### 5.1.2 处理流程

1. 用户登录时，系统获取用户所属shop_id，存入租户上下文
2. 所有数据访问自动携带shop_id条件
3. 数据插入时自动注入shop_id
4. shop_id不一致时拒绝访问

#### 5.1.3 涉及数据

| 数据项 | 数据来源 | 数据用途 | 说明 |
|-------|---------|---------|------|
| shop_id | 用户登录上下文 | 数据隔离标识 | 所有业务表都有shop_id字段 |
| 租户上下文 | Spring Filter / 拦截器 | 请求级别数据传递 | ThreadLocal存储 |

#### 5.1.4 业务规则

- **规则1：强制隔离**
  - 规则描述：所有数据查询必须携带shop_id，无shop_id的请求拒绝访问
  - 约束说明：基础设施层统一拦截，业务代码无感知

- **规则2：数据注入**
  - 规则描述：所有数据插入自动带上当前租户的shop_id
  - 约束说明：不允许业务代码手动指定shop_id

#### 5.1.5 异常处理

| 异常场景 | 处理方式 | 错误提示/日志 |
|---------|---------|--------------|
| 未登录访问 | 拒绝访问，重定向登录页 | 无权限访问 |
| shop_id为空 | 拒绝访问 | 租户上下文缺失 |
| 跨店铺访问 | 拒绝访问 | 数据不属于当前店铺 |

#### 5.1.6 与其他功能的关联

- **上游依赖：**无
- **下游影响：**所有业务模块依赖此基础设施

### 5.2 邀请码服务

#### 5.2.1 功能概述

**触发方式：**API调用
**触发时机：**生成邀请码、验证邀请码
**功能职责：**提供邀请码的生成、验证和状态管理能力，支持店铺用户和管理员生成邀请码

#### 5.2.2 处理流程

**生成邀请码：**
1. 调用生成接口，传入目标账号类型（店铺用户/普通用户）
2. 生成唯一邀请码（UUID或随机字符串）
3. 存储邀请码、目标类型、创建人、创建时间、状态（未使用/已使用/已失效）
4. 返回邀请码

**验证邀请码：**
1. 用户注册时传入邀请码
2. 验证邀请码存在、状态为"未使用"、未过期
3. 验证通过后标记为"已使用"
4. 返回验证结果

#### 5.2.3 涉及数据

| 数据项 | 数据来源 | 数据用途 | 说明 |
|-------|---------|---------|------|
| 邀请码 | 系统生成 | 唯一标识 | UUID，永久有效 |
| 目标账号类型 | 调用时传入 | 限制使用范围 | 店铺用户/普通用户 |
| 创建人 | 当前登录用户 | 记录来源 | 系统内置字段 |
| 状态 | 系统维护 | 管理生命周期 | 未使用/已使用/已失效 |

#### 5.2.4 业务规则

- **规则1：邀请码永久有效**
  - 规则描述：邀请码生成后永久有效，不过期
  - 约束说明：无时间限制

- **规则2：重新生成后旧码失效**
  - 规则描述：同一账号类型重新生成邀请码后，旧邀请码立即失效
  - 约束说明：已在注册/审核中的账号继续可用

- **规则3：邀请码唯一性**
  - 规则描述：同一账号类型的邀请码在同一时间只有1个有效
  - 约束说明：生成新码时自动使旧码失效

#### 5.2.5 异常处理

| 异常场景 | 处理方式 | 错误提示/日志 |
|---------|---------|--------------|
| 邀请码不存在 | 返回验证失败 | 日志记录 |
| 邀请码已使用 | 返回验证失败 | 日志记录 |
| 邀请码已失效 | 返回验证失败 | 日志记录 |

#### 5.2.6 与其他功能的关联

- **上游依赖：**用户认证
- **下游影响：**店铺入驻流程、普通用户注册流程

### 5.3 消息通知服务

#### 5.3.1 功能概述

**触发方式：**API调用 / 定时任务
**触发时机：**订单超时、审核结果、密码重置等场景触发
**功能职责：**提供站内消息和短信通知能力，支持多渠道通知

#### 5.3.2 处理流程

**消息发送：**
1. 业务模块调用消息发送接口
2. 消息服务存储消息记录（站内消息）
3. 如需短信通知，调用短信网关发送
4. 返回发送结果

**站内消息查看：**
1. 用户进入消息列表页
2. 查询当前用户的消息记录
3. 分页返回消息列表

#### 5.3.3 涉及数据

| 数据项 | 数据来源 | 数据用途 | 说明 |
|-------|---------|---------|------|
| 消息类型 | 调用时传入 | 分类显示 | 订单超时/审核结果/密码重置等 |
| 接收人 | 调用时传入 | 消息目标 | 用户ID |
| 消息内容 | 调用时传入 | 消息主体 | 根据类型自动组装 |
| 是否已读 | 系统维护 | 状态标识 | 未读/已读 |
| 发送时间 | 系统生成 | 时间记录 | 自动记录 |

#### 5.3.4 业务规则

- **规则1：订单超时提醒**
  - 规则描述：订单状态为"已下单"超过7天未确认，自动发送短信提醒店铺用户
  - 触发条件：每天凌晨2点定时扫描
  - 约束说明：同一订单仅提醒一次

- **规则2：审核结果通知**
  - 规则描述：账号/店铺审核完成后，发送短信通知申请人
  - 触发条件：审核操作完成后
  - 约束说明：仅在有手机号时发送

- **规则3：密码重置通知**
  - 规则描述：密码被重置后，发送短信通知用户
  - 触发条件：密码重置操作完成后
  - 约束说明：发送新密码到注册手机号

#### 5.3.5 异常处理

| 异常场景 | 处理方式 | 错误提示/日志 |
|---------|---------|--------------|
| 短信发送失败 | 记录失败日志，重试3次 | 发送失败短信：网络异常 |
| 无手机号 | 跳过短信，仅发站内消息 | 日志记录 |
| 消息存储失败 | 返回错误 | 事务回滚 |

#### 5.3.6 与其他功能的关联

- **上游依赖：**订单管理、用户管理、店铺管理
- **下游影响：**无
