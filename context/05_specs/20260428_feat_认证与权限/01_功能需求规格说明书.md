## 1. 概述

### 1.1 功能背景

多店铺积分商城系统的身份认证和权限控制基础模块。系统分为三种角色（管理员、店铺用户、普通用户），通过账号密码登录，支持邀请码注册。

### 1.2 本功能业务目标

1. 实现三种角色的账号密码登录
2. 支持邀请码注册（需审核）
3. 管理员可管理所有用户，店铺用户只能管理本店普通用户
4. 支持用户冻结、删除、密码重置

---

## 2. 角色与权限矩阵

### 2.1 数据权限

| 角色 | 数据范围 | 说明 |
|------|---------|------|
| 管理员 | 全部用户 | 可管理所有用户 |
| 店铺用户 | 本店用户 | 只能管理本店（shop_id）普通用户 |
| 普通用户 | 仅自己 | 只能操作自己的账号 |

### 2.2 功能权限

| 功能 | 管理员 | 店铺用户 | 普通用户 |
|------|:------:|:-------:|:-------:|
| 登录 | √ | √ | √ |
| 注册（邀请码） | - | - | √ |
| 查看用户列表 | 全部 | 本店 | - |
| 审核用户 | √（店铺用户注册申请） | √（普通用户注册申请） | - |
| 冻结/解冻用户 | √ | - | - |
| 删除用户 | √ | √（本店） | - |
| 重置密码 | √ | √（本店） | - |
| 生成邀请码 | √（创建店铺用户） | √（创建普通用户） | - |

## 4. 页面功能详细说明

### 4.1 登录页

**涉及字段：**
| 字段 | 类型 | 校验 |
|------|------|------|
| 用户名 | 文本 | 4-20字符 |
| 密码 | 密码 | 8-20位字母+数字 |

**功能：账号密码登录**
- 验证用户名+密码
- 验证账号状态（已冻结/待审核禁止登录）
- 成功生成Token跳转首页，失败提示"用户名或密码错误"

**业务规则：**
- Token有效期默认7天（168小时），可配置精确到小时
- 登录账号状态必须为"已通过"方可登录
- 待审核、已拒绝、已冻结状态均禁止登录

---

### 4.2 注册页

**涉及字段：**
| 字段 | 类型 | 校验 |
|------|------|------|
| 邀请码 | 文本 | 必填 |
| 用户名 | 文本 | 4-20字符，字母开头 |
| 昵称 | 文本 | 2-20字符 |
| 密码 | 密码 | 8-20位字母+数字 |
| 确认密码 | 密码 | 与密码相同 |

**功能：**
1. 验证邀请码有效性 → 填写注册信息 → 提交
2. 提交后账号状态为"待审核"，提示"请等待审核"

**业务规则：**
- 邀请码验证通过后才能填写注册信息
- 用户名唯一性校验

---

### 4.3 用户管理

**涉及字段：**
| 字段 | 类型 | 说明 |
|------|------|------|
| 关键词 | 文本 | 搜索用户名/昵称 |
| 角色 | 下拉 | 管理员/店铺用户/普通用户 |
| 状态 | 下拉 | 待审核/已通过/已拒绝/已冻结 |

**显示字段：** 用户ID、用户名、昵称、角色、所属店铺、账号状态、创建时间

**功能：**

| 功能 | 按钮位置 | 说明 |
|------|---------|------|
| 查看列表 | 页面加载 | 分页展示，管理员看全部，店铺用户看本店 |
| 生成邀请码 | 页面级 | 生成邀请码用于注册新用户，需填写注册人角色 |

**生成邀请码涉及字段：**
| 字段 | 类型 | 校验 |
|------|------|------|
| 角色 | 下拉 | 管理员生成：店铺用户；店铺用户生成：普通用户 |

**审核用户涉及字段：**
| 字段 | 类型 | 校验 |
|------|------|------|
| 拒绝理由 | 文本 | 必填，1-200字符 |

**功能按钮：**
| 功能 | 按钮位置 | 说明 |
|------|---------|------|
| 审核用户 | 行级（待审核） | 通过/拒绝，拒绝需填理由 |
| 冻结用户 | 行级（已通过） | 状态变为已冻结，用户无法登录 |
| 解冻用户 | 行级（已冻结） | 状态恢复为已通过 |
| 删除用户 | 行级 | 逻辑删除，已登录的清下线 |
| 重置密码 | 行级 | 随机生成密码，弹框显示 |

**审核流程：**
1. 管理员生成邀请码 → 他人使用邀请码注册（店铺用户角色） → 管理员审核
2. 店铺用户生成邀请码 → 他人使用邀请码注册（普通用户角色） → 店铺用户审核
3. 审核拒绝时用户可重新发起注册申请

**业务规则：**
- 管理员：可管理所有用户，可冻结/解冻
- 店铺用户：只能管理本店普通用户，无冻结权限
- 删除已冻结用户时直接清除登录态，无需解冻

## 5. 数据状态定义

### 账号状态

| 状态 | 说明 | 可转换 |
|------|------|--------|
| 待审核 | 等待审核 | 已通过、已拒绝 |
| 已通过 | 可正常登录 | 已冻结 |
| 已拒绝 | 可重新注册 | 待审核 |
| 已冻结 | 无法登录 | 已通过 |

### 邀请码状态

| 状态 | 说明 |
|------|------|
| 有效 | 可用于注册 |
| 已使用 | 已被使用 |
| 已失效 | 重新生成后失效 |
