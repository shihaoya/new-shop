## 1. 项目背景
多租户电商平台**，以**积分作为唯一货币**的商品兑换系统。每个店铺拥有
多店铺积分商城系统是一个**独立的数据和运营管理能力，平台方（管理员）可俯瞰所有店铺运营数据。

**业务性质**：
- 业务类型：多店铺积分商城系统
- 行业分类：通用电商（积分兑换场景）
- 目标用户：企业/品牌方运营多店铺积分商城

**多租户架构**：
- **平台层**：管理员（平台方），全局数据，可查看所有店铺数据
- **店铺层**：店铺用户（每个店铺的管理员），本店数据完全独立
- **用户层**：普通用户（店铺的消费者），只能访问所属店铺的数据

**核心资源与能力**：
- 积分体系：店铺用户手动维护积分，积分=货币，商品只能由积分兑换
- 商品分类：系统内置分类（虚拟/实物）与商家自定义分类平级独立
- 订单流程：已下单→已确认→已发货→已完成，支持超时提醒
- 数据隔离：表级别隔离，通过shop_id区分

## 2. 项目目标

**核心目标与管控诉求**：
- 为企业/品牌方提供多店铺独立运营的积分兑换平台
- 支持店铺级别数据隔离，确保各店铺数据安全
- 建立完善的账号审核机制，保障平台秩序
- 提供便捷的订单处理和积分管理能力

**内外部约束**：
- 积分无过期机制，永不过期
- 店铺间数据完全隔离，不可跨店查询
- 实物商品需要物流（买家填写收货信息，卖家填写快递信息）
- 店铺用户账号与店铺是独立概念，需先审核账号再审核店铺

## 4. 功能模块详细说明

### 4.1 管理员-仪表盘

#### 4.1.1 模块业务目标

为平台管理员提供全局数据监控能力，实时掌握所有店铺的运营状况，包括店铺数量、用户总数、订单总量、积分发放统计等关键指标。

#### 4.1.2 模块业务流程

管理员进入仪表盘后，系统实时展示全局数据概览，包括所有店铺数量、用户总数、订单总量、积分发放统计。

#### 4.1.3 功能列表

| 功能 | 子功能 | 说明 |
|:-----|:-------|:-----|
| 全局数据统计 | 查看所有店铺数量 | 管理员在仪表盘查看平台所有店铺总数 |
| 全局数据统计 | 查看用户总数 | 管理员在仪表盘查看所有店铺的注册用户总数 |
| 全局数据统计 | 查看订单总量 | 管理员在仪表盘查看平台所有订单数量 |
| 全局数据统计 | 查看积分发放统计 | 管理员在仪表盘查看平台积分发放情况（实时统计） |

#### 4.1.4 操作人员定义

**管理员（平台层）**：平台超级管理员，系统初始化时内置第一个账号，负责全局数据监控、店铺审核管理、跨店铺用户管理。

#### 4.1.5 对接的外部系统

无（管理员模块不涉及外部系统对接）

#### 4.1.6 模块边界和依赖关系

本模块为系统内置统计模块，不依赖其他业务模块，被店铺用户仪表盘、订单模块间接依赖。

---

### 4.2 管理员-店铺管理

#### 4.2.1 模块业务目标

为平台管理员提供店铺审核与管理能力，确保店铺合法合规运营。包括店铺申请审核、违规店铺冻结/解冻、店铺列表查看等功能。

#### 4.2.2 模块业务流程

1. 管理员查看待审核店铺申请列表
2. 管理员审核店铺申请（通过/拒绝）
3. 管理员可对违规店铺执行冻结操作
4. 管理员可对已冻结店铺执行解冻操作
5. 管理员可查看所有店铺列表及详情

| 功能 | 子功能 | 说明 |
|:-----|:-------|:-----|
| 店铺审核 | 查看店铺申请列表 | 管理员查看所有待审核的店铺申请 |
| 店铺审核 | 审核店铺申请 | 管理员审核店铺申请，通过或拒绝申请 |
| 店铺管理 | 冻结店铺 | 管理员对违规店铺执行冻结操作，冻结后店铺商品自动下架，店铺用户不能操作商品 |
| 店铺管理 | 解冻店铺 | 管理员对已冻结店铺执行解冻操作 |
| 店铺管理 | 查看店铺列表 | 管理员查看所有店铺的列表及状态 |
| 店铺管理 | 查看店铺详情 | 管理员查看单个店铺的详细信息 |
| 店铺管理 | 查看历史申请记录 | 管理员查看店铺的历史申请/审核/冻结/解冻记录（时间线形式） |

#### 4.2.4 操作人员定义

**管理员（平台层）**：拥有店铺审核、冻结/解冻、查看店铺列表和详情的权限。

#### 4.2.5 对接的外部系统

无

#### 4.2.6 模块边界和依赖关系

本模块管理店铺状态，影响店铺用户的功能可用性（店铺「申请中」或「已冻结」时不能操作上架商品）。

---

### 4.3 管理员-用户管理

#### 4.3.1 模块业务目标

为平台管理员提供跨店铺用户账号管理能力，包括查看所有店铺用户和普通用户、强制重置任何用户密码等。

#### 4.3.2 模块业务流程

1. 管理员查看所有店铺用户列表（跨店铺）
2. 管理员查看所有普通用户列表（跨店铺）
3. 管理员冻结/解冻任意用户
4. 管理员选择任意用户执行密码强制重置

#### 4.3.3 功能列表

| 功能 | 子功能 | 说明 |
|:-----|:-------|:-----|
| 用户账号管理 | 查看店铺用户列表 | 管理员跨店铺查看所有店铺用户账号 |
| 用户账号管理 | 查看普通用户列表 | 管理员跨店铺查看所有普通用户账号 |
| 用户账号管理 | 编辑用户 | 管理员编辑用户昵称和状态（启用/禁用） |
| 用户账号管理 | 冻结/解冻用户 | 管理员冻结/解冻任何用户（店铺用户/普通用户） |
| 用户账号管理 | 强制重置密码 | 管理员强制重置任何用户（店铺用户/普通用户）的密码 |

#### 4.3.4 操作人员定义

**管理员（平台层）**：可查看所有用户列表、冻结/解冻任何用户、重置任何用户密码。

#### 4.3.5 对接的外部系统

无

#### 4.3.6 模块边界和依赖关系

本模块仅查看和管理用户账号，不涉及积分操作（积分由店铺用户维护）。

---

### 4.4 管理员-个人中心

#### 4.4.1 模块业务目标

为平台管理员提供个人账号信息管理和邀请码生成能力，支持管理员修改个人信息和生成邀请码邀请店铺用户。

#### 4.4.2 模块业务流程

1. 管理员修改个人信息（昵称、手机号等）
2. 管理员修改登录密码
3. 管理员生成邀请码，用于邀请店铺用户注册（需审核）

#### 4.4.3 功能列表

| 功能 | 子功能 | 说明 |
|:-----|:-------|:-----|
| 个人信息管理 | 修改个人信息 | 管理员修改自己的账号信息（如昵称） |
| 个人信息管理 | 修改密码 | 管理员修改自己的登录密码 |
| 邀请码管理 | 生成邀请码 | 管理员生成永久有效的邀请码用于邀请店铺用户（需审核） |
| 邀请码管理 | 重新生成邀请码 | 管理员重新生成邀请码，新码生成后旧码立即失效（已在注册/审核中的账号继续可用） |
| 邀请码管理 | 复制邀请码 | 管理员复制当前有效的邀请码到剪贴板 |

#### 4.4.4 操作人员定义

**管理员（平台层）**：管理自身账号信息和生成邀请码。

#### 4.4.5 对接的外部系统

无

#### 4.4.6 模块边界和依赖关系

本模块为管理员提供自服务能力，邀请码功能对接后续的店铺用户注册流程。

---

### 4.5 店铺用户-仪表盘

#### 4.5.1 模块业务目标

为店铺用户提供本店运营数据概览，包括本店用户数、商品数、订单数、积分余额等统计信息，帮助店铺运营者快速了解店铺状态。

#### 4.5.2 模块业务流程

店铺用户登录后进入仪表盘，系统展示本店实时数据：店铺用户数、普通用户数、商品数（上架/下架）、订单数（按状态分类）、积分余额汇总。

#### 4.5.3 功能列表

| 功能 | 子功能 | 说明 |
|:-----|:-------|:-----|
| 本店数据统计 | 查看店铺用户数 | 店铺用户在仪表盘查看本店店铺用户数量 |
| 本店数据统计 | 查看普通用户数 | 店铺用户在仪表盘查看本店普通用户数量 |
| 本店数据统计 | 查看商品数 | 店铺用户在仪表盘查看本店商品数量（上架/下架） |
| 本店数据统计 | 查看订单数 | 店铺用户在仪表盘查看本店订单数量（按状态分类） |
| 本店数据统计 | 查看积分余额统计 | 店铺用户在仪表盘查看本店积分余额汇总 |

#### 4.5.4 操作人员定义

**店铺用户（店铺层）**：查看本店数据统计，仅限本店数据。

#### 4.5.5 对接的外部系统

无

#### 4.5.6 模块边界和依赖关系

本模块聚合商品管理、用户管理、订单管理的数据，被动依赖这些模块的数据产出。

---

### 4.6 店铺用户-商品分类管理

#### 4.6.1 模块业务目标

为店铺用户提供本店商品分类管理能力，支持创建、编辑、删除、批量删除、查看商品分类，便于对本店商品进行分组管理。

#### 4.6.2 模块业务流程

1. 店铺用户查看本店所有分类列表（按排序值升序排列）
2. 店铺用户新建分类（填写名称、排序值），系统校验名称唯一性
3. 店铺用户编辑已有分类名称和排序值
4. 店铺用户删除无商品关联的分类
5. 店铺用户批量选择分类后批量删除（系统逐一检查商品关联）

#### 4.6.3 功能列表

| 功能 | 子功能 | 说明 |
|:-----|:-------|:-----|
| 分类管理 | 查看分类列表 | 店铺用户查看本店所有分类列表（按排序值升序） |
| 分类管理 | 新建分类 | 店铺用户创建本店分类（填写名称/排序值），系统校验名称唯一性 |
| 分类管理 | 编辑分类 | 店铺用户修改分类名称和排序值 |
| 分类管理 | 删除分类 | 店铺用户删除无商品关联的分类，有关联时提示无法删除 |
| 分类管理 | 批量删除 | 店铺用户批量选择分类后删除，系统逐一检查商品关联，有关联的跳过并提示 |

---

### 4.7 店铺用户-商品管理

#### 4.7.1 模块业务目标

为店铺用户提供完整的商品生命周期管理能力，支持搜索、筛选、创建、编辑、上架、下架、删除、查看等操作，区分虚拟商品和实物商品。

#### 4.7.2 模块业务流程

1. 店铺用户搜索本店商品（关键词模糊搜索）
2. 店铺用户筛选商品（按分类、上架状态）
3. 店铺用户创建商品（填写名称/图片/积分/库存/分类/描述/类别）
4. 店铺用户编辑已有商品信息
5. 店铺用户将下架商品上架销售
6. 店铺用户将上架商品下架
7. 店铺用户删除本店商品（逻辑删除，店铺冻结时不可删除）
8. 店铺用户查看单个商品详情（店铺冻结时仅可查看）

#### 4.7.3 功能列表

| 功能 | 子功能 | 说明 |
|:-----|:-------|:-----|
| 商品管理 | 搜索商品 | 店铺用户通过关键词搜索本店商品 |
| 商品管理 | 筛选商品 | 店铺用户按商品分类、上架状态筛选商品 |
| 商品管理 | 创建商品 | 店铺用户填写商品信息（名称/图片/积分/库存/分类/描述/类别）创建商品 |
| 商品管理 | 编辑商品 | 店铺用户修改已有商品的信息 |
| 商品管理 | 上架商品 | 店铺用户将下架商品上架销售 |
| 商品管理 | 下架商品 | 店铺用户将上架商品下架 |
| 商品管理 | 删除商品 | 店铺用户删除本店商品（逻辑删除），店铺状态为「已冻结」时不可删除 |
| 商品管理 | 查看商品详情 | 店铺用户查看单个商品的详细信息，店铺状态为「已冻结」时仅可查看不可编辑 |

#### 4.7.4 操作人员定义

**店铺用户（店铺层）**：管理本店商品，但需店铺状态为「已通过」才能上架商品。

#### 4.7.5 对接的外部系统

**文件存储服务**：商品图片上传存储（当前为本地存储，预留对象存储扩展接口）。

#### 4.7.6 模块边界和依赖关系

本模块依赖商品分类管理模块（选择分类），被普通用户商品列表模块依赖。

---

### 4.8 店铺用户-用户管理

#### 4.8.1 模块业务目标

为店铺用户提供本店普通用户管理能力，包括用户查看、创建、审核、冻结、删除、积分维护、密码重置等完整用户生命周期管理。

#### 4.8.2 模块业务流程

1. 店铺用户查看本店普通用户列表
2. 店铺用户新建普通用户账号（设置初始积分），默认状态为已通过
3. 店铺用户编辑普通用户昵称
4. 店铺用户通过Excel批量导入普通用户（全成功或全失败），默认状态为已通过
5. 店铺用户审核通过/拒绝邀请码注册的普通用户（待审核状态）
6. 店铺用户冻结/解冻本店普通用户
7. 店铺用户删除本店普通用户（逻辑删除）
8. 店铺用户增加/扣除普通用户积分（必须填写操作理由）
9. 店铺用户查看普通用户积分变动记录
10. 店铺用户重置本店普通用户密码

#### 4.8.3 功能列表

| 功能 | 子功能 | 说明 |
|:-----|:-------|:-----|
| 普通用户管理 | 查看普通用户列表 | 店铺用户查看本店所有普通用户，支持用户名搜索和状态筛选 |
| 普通用户管理 | 新建普通用户 | 店铺用户直接创建普通用户账号（设置用户名/昵称/初始积分），默认状态为已通过，系统生成随机密码并弹框展示 |
| 普通用户管理 | 编辑普通用户 | 店铺用户修改普通用户昵称 |
| 普通用户管理 | 批量导入用户 | 店铺用户通过Excel批量导入普通用户（全成功或全失败），默认状态为已通过，导入成功显示用户名和新密码 |
| 普通用户管理 | 审核普通用户 | 店铺用户审核邀请码注册的普通用户申请，通过后用户可登录，拒绝后用户不可登录 |
| 用户冻结 | 冻结普通用户 | 店铺用户冻结本店普通用户（冻结后用户无法登录） |
| 用户冻结 | 解冻普通用户 | 店铺用户解冻已冻结的本店普通用户 |

## 5. Feature拆分推荐

按推荐开发顺序排列，作为后续拆分Feature的参考。每个Feature控制在6个功能点以内，超过则需拆分。

**功能点计数说明**：06_功能列表.md中"功能模块"列的每个唯一"功能"计为1个功能点。例如"管理员-仪表盘"下有"全局数据统计"1个功能，则功能点数量为1。

| 序号 | Feature名称 | 包含功能 | Feature类型 | 依赖Feature |
|:----:|:------------|:---------|:------------|:------------|
| 1 | 系统基础设施 | 数据隔离、文件存储（文件上传/预览/删除/下载） | 基础设施 | - |
| 2 | 认证与权限 | 登录注册、邀请码、用户管理（查看/创建/审核/冻结/解冻/删除）、密码重置 | 基础设施 | - |
| 3 | 消息通知 | 站内消息通知（超时提醒/审核通知/密码通知）、消息查看（列表/详情/标记已读/批量标记）、消息类型配置 | 通用服务 | Feature1 |
| 4 | 管理员-店铺管理 | 店铺审核、冻结/解冻、查看列表/详情 | 业务功能 | Feature1, Feature2 |
| 5 | 管理员-用户管理 | 跨店铺用户管理、冻结/解冻、密码重置 | 业务功能 | Feature1, Feature2 |
| 6 | 店铺用户-我的店铺 | 店铺申请/状态/重新申请、邀请码管理 | 业务功能 | Feature2, Feature4 |
| 7 | 店铺用户-商品分类管理 | 分类CRUD | 业务功能 | Feature6 |
| 8 | 店铺用户-商品管理 | 商品CRUD、上架下架 | 业务功能 | Feature6, Feature7 |
| 9 | 店铺用户-用户管理 | 普通用户管理（查看/创建/审核/冻结/解冻/删除）、积分维护、密码重置 | 业务功能 | Feature6 |
| 10 | 普通用户-地址簿 | 地址簿CRUD、设默认 | 业务功能 | Feature1 |
| 11 | 普通用户-商品兑换 | 商品浏览、兑换、收货信息 | 业务功能 | Feature8, Feature10 |
| 12 | 店铺用户-订单管理 | 订单处理、发货、完成、关闭 | 业务功能 | Feature8 |
| 13 | 普通用户-订单管理 | 订单查看、关闭 | 业务功能 | Feature11, Feature12 |

**Feature类型说明**：
- **基础设施**：认证、权限、日志、配置等全局能力
- **通用服务**：文件存储、消息通知、定时任务等复用服务
- **业务功能**：具体业务功能
- **业务组件**：特定业务场景的技术实现（报表、流程引擎等）
- **外部集成**：与外部系统的对接层

**拆分约束**：
- 每个Feature的功能点数量 ≤ 6个
- 超过6个功能点需拆分为多个Feature

## 6. 术语表

| 术语 | 定义 |
|:-----|:-----|
| 多租户 | 一种软件架构模式，多个租户共享同一个系统实例，但数据完全隔离。在本系统中，每个店铺为一个租户，拥有独立的数据。 |
| 积分商城 | 以积分作为唯一货币的商品兑换系统，用户通过积累的积分来兑换商品，不支持现金支付。 |
| 店铺用户 | 每个店铺的管理员账号，负责本店的商品管理、订单处理、用户积分维护等运营工作。账号与店铺是独立概念。 |
| 普通用户 | 店铺的消费者，通过积分兑换商品的用户群体。每个普通用户只属于一个店铺。 |
| 商家自定义分类 | 店铺用户自行创建的商品分类，用于对本店商品进行分组管理。 |
| 系统内置分类 | 系统预置的商品分类（如：虚拟、实物等），店铺用户不可编辑，仅可选择。 |
| 逻辑删除 | 一种数据删除方式，不是真正从数据库中删除数据，而是通过修改状态字段使数据不可见。 |
| 表级别隔离 | 多租户数据隔离的一种实现方式，所有租户的数据存在同一张表中，通过 shop_id 字段区分数据归属。 |
| 单用户最大兑换个数 | 商品的可重复兑换限制设置。默认0表示无限制，非0表示同一用户对该商品的最高兑换次数。 |
| 积分操作记录 | 店铺用户增加或扣除普通用户积分时，必须填写的操作原因说明，用于积分变动的可追溯。 |
| 订单超时提醒 | 订单状态为「已下单」超过7天未确认时，系统自动发送提醒通知店铺用户处理。 |
| 店铺冻结 | 管理员对店铺执行的强制封停操作。店铺冻结后，店铺用户无法操作商品和用户，商品自动下架。 |
| 邀请码 | 用于替代直接注册/创建流程的安全凭证。唯一对应一个目标账号类型（店铺用户/普通用户），生成后永久有效，重新生成后旧码立即失效。 |
| 强制重置密码 | 管理员或店铺用户对用户账号执行的密码重置操作。系统随机生成新密码，弹框显示后由执行者线下告知用户。 |
| shop_id | 表级别隔离中用于标识数据所属店铺的字段。通过 shop_id 过滤确保店铺间数据完全隔离。 |
| 地址簿 | 普通用户管理的收货地址集合，用于实物商品下单时快速填写收货信息。最多5个地址。 |
| 默认地址 | 普通用户设置的优先使用的收货地址，下单时自动带入。仅能设置1个。 |
| 实物商品 | 需要物流配送的实体商品，下单时需填写收货人、手机、地址。 |
| 虚拟商品 | 电子凭证类商品（如会员卡、点卡），无需物流，下单后直接发放。 |
