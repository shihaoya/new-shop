## 一、业务性质

### 1.1 业务类型

| 属性 | 说明 |
|------|------|
| 业务类型 | 多店铺积分商城系统 |
| 行业分类 | 通用电商（积分兑换场景） |
| 目标用户 | 企业/品牌方运营多店铺积分商城 |

### 1.2 核心定位

积分商城系统是一个**多租户电商平台**，以**积分作为唯一货币**的商品兑换系统。每个店铺拥有独立的数据和运营管理能力，平台方（管理员）可俯瞰所有店铺运营数据。

### 1.3 多租户架构

| 层级 | 说明 | 数据隔离 |
|------|------|----------|
| 平台层 | 管理员（平台方） | 全局数据，可查看所有店铺数据 |
| 店铺层 | 店铺用户（每个店铺的管理员） | 本店数据完全独立 |
| 用户层 | 普通用户（店铺的消费者） | 只能访问所属店铺的数据 |

## 二、干系人与角色

### 2.1 角色定义

| 角色 | 说明 | 账号创建方式 |
|------|------|--------------|
| 管理员 | 平台超级管理员，系统初始化时内置第一个账号，可新增任何用户 | 系统内置（初始账号） |
| 店铺用户 | 每个店铺的管理员账号（账号与店铺是独立概念） | 管理员创建 或 自行注册（需审核账号） |
| 普通用户 | 店铺的消费者 | 店铺用户注册或自行注册邀请码方式（需审核） |

### 2.2 层级归属关系

```
管理员（平台层）
    └── 店铺用户 A（店铺层）
            └── 普通用户 A1, A2, ...（用户层）
    └── 店铺用户 B（店铺层）
            └── 普通用户 B1, B2, ...（用户层）
```

### 2.3 账号创建与店铺申请流程

**完整流程（5个步骤）**：

```
1. 店铺用户注册 → 管理员审核账号（控制登录权限）
    ↓ 通过后
2. 店铺用户登录（立刻可以登录）
    ↓
3. 店铺用户注册店铺（填写店铺名称、logo、简介等）
    ↓
4. 管理员审核店铺（控制经营权限）
    ↓ 通过
5. 店铺用户上架商品
```

**2.3.1 账号注册流程**

```
店铺用户 → 提交注册申请 → 管理员审核 → 通过（可登录）/ 拒绝
```

**2.3.2 店铺申请流程**

```
店铺用户登录后 → 填写店铺申请信息 → 提交申请 → 管理员审核
    ↓ 拒绝
店铺用户可重新填写理由再次申请
    ↓ 通过
店铺状态变为「已通过」，店铺用户可正常上架商品

后续：管理员可冻结店铺 → 店铺状态变为「已冻结」并且下架所有商品，已经下单的商品可继续操作不受影响
    ↓
店铺冻结后，店铺用户需重新填写理由审核
```

**2.3.3 普通用户创建**

```
1. 店铺用户 → 生成邀请码 → 普通用户注册（填邀请码） → 店铺用户审核 → 通过/拒绝
2. 店铺用户 → 直接创建/excel批量创建 → 系统随机生成密码 → 线下分发账号
```

### 2.4 店铺状态

| 状态 | 说明 | 店铺用户权限 |
|------|------|--------------|
| 未申请 | 刚注册账号没有店铺信息 | 仅可查看，不可操作商品 |
| 申请中 | 等待管理员审核 | 仅可查看，不可操作商品 |
| 已通过 | 管理员审核通过 | 可正常管理商品、订单、用户积分 |
| 已冻结 | 管理员冻结了店铺 | 无法操作，需要重新申请 |

### 2.5 密码重置机制

| 执行者 | 对象 | 方式 |
|--------|------|------|
| 管理员 | 任何用户（店铺用户、普通用户） | 强制重置密码，随机生成，弹框显示，由管理员线下告知 |
| 店铺用户 | 本店普通用户 | 强制重置密码，随机生成，弹框显示，由店铺用户线下告知 |

## 三、功能模块（按角色）

### 3.1 管理员（平台层）

| 模块 | 功能 |
|------|------|
| 仪表盘 | 查看全局数据：所有店铺数量、用户总数、订单总量、积分发放统计（实时统计） |
| 店铺管理 | 审核店铺申请（通过/拒绝）/ 冻结/解冻店铺 / 查看店铺列表 |
| 用户管理 | 查看/管理所有店铺用户和普通用户（跨店铺），可强制重置任何用户密码 |
| 个人中心 | 修改个人信息 |

### 3.2 店铺用户（店铺层）

> **注意**：店铺用户需要店铺「已通过」审核后才能上架商品、维护用户等功能。店铺被冻结后功能受限。

| 模块 | 功能 | 店铺状态限制 |
|------|------|--------------|
| 仪表盘 | 查看本店数据：店铺用户数、普通用户数、商品数、订单数、积分余额统计
| 商品分类管理 | 创建/编辑/删除本店商品分类
| 商品管理 | 创建/编辑/上架/下架/删除本店商品，设置积分价格、库存、单用户最大兑换个数（卡片/表格两种展示形式） | 仅「已通过」可上架商品 |

#### 商品字段说明

| 字段 | 说明 | 必填 |
|------|------|------|
| 名称 | 商品名称 | 必填 |
| 图片 | 商品主图 | 必填 |
| 积分价格 | 兑换所需积分（纯积分，无现金） | 必填 |
| 库存 | 商品可兑换数量，下单时扣减 | 必填 |
| 商品分类 | 选择所属分类（系统内置分类或商家自定义分类） | 必填 |
| 状态 | 上架/下架 | 必填 |
| 描述 | 商品详细描述 | 选填 |
| 单用户最大兑换个数 | 同一用户可重复兑换次数，默认0=无限制 | 选填 |
| 用户管理 | 查看/管理本店普通用户，手动维护用户积分（增加/扣除），可强制重置本店用户密码 | 仅「已通过」可用 |
| 订单管理 | 查看/处理本店订单（卡片/表格两种展示形式）
| 个人中心 | 修改个人信息 | 无限制 |
| 我的店铺 | 申请店铺（填写店铺名称、logo、简介）/ 查看店铺状态 / 被拒绝后可重新申请 / 被冻结后需重新申请 | 无限制 |

### 3.3 普通用户（用户层）

| 模块 | 功能 |
|------|------|
| 商品列表 | 浏览本店商品（卡片/表格两种展示形式） |
| 我的订单 | 查看个人订单历史 |
| 个人中心 | 修改个人信息、查看积分余额 |

## 四、商品分类体系

### 4.1 分类类型

| 分类类型 | 说明 | 管理方式 |
|----------|------|----------|
| 系统内置分类 | 系统预置的分类（如：新品、热销、推荐等），店铺用户不可编辑 | 系统管理 |
| 商家自定义分类 | 店铺用户自行创建的分类 | 店铺用户管理 |

> **说明**：系统内置分类与商家自定义分类是**平级独立**的关系，无父子归属。商品在创建时选择使用哪一类分类。

```
系统内置分类（店铺用户不可编辑）

    └── 实物

商家自定义分类（店铺用户自行管理）
    ├── 游戏点卡
    ├── 会员卡
    ├── 电子券
    ├── 数码配件
    ├── 家居用品
    └── 食品生鲜
```

## 五、积分体系

### 5.1 积分规则

| 规则 | 说明 |
|------|------|
| 积分获取 | 由店铺用户**手动维护**（线下统计后录入） |
| 积分用途 | 积分=货币，商品只能由积分兑换 |
| 积分查询 | 普通用户可在个人中心查看积分余额 |
| 积分管理 | 店铺用户可增加/扣除用户积分 |
| 积分操作记录 | 增加/扣除积分时**必须填写操作理由** |
| 积分过期 | **无过期机制**，积分永不过期 |
| 普通用户初始积分 | 店铺用户开通时可传入初始值；自行注册默认0积分 |

### 5.2 积分与订单

| 场景 | 说明 |
|------|------|
| 商品价格 | 商品设置积分价格（纯积分，无现金） |
| 下单支付 | 用户使用积分兑换商品 |
| 订单关闭 | 积分退还用户账户 |
| 订单完成后 | 积分从用户账户扣除 |

## 六、订单流程

### 6.0 超时提醒机制

> 订单「已下单」状态超过 **7天** 未确认，系统发送提醒（不自动关闭订单）。

### 6.1 订单状态

| 状态 | 说明 | 可关闭操作 |
|------|------|------------|
| 已下单 | 订单已创建，积分已扣除，等待商家确认 | 用户可关闭、店铺用户可关闭 |
| 已确认 | 商家已确认订单，等待发货 | 仅店铺用户可关闭 |
| 已发货 | 商家已发货，配送中 | 仅店铺用户可关闭 |
| 已完成 | 订单完成，不可关闭 | 无 |
| 已关闭 | 订单已关闭 | 无（仅可查看） |

### 6.2 订单关闭规则

| 执行者 | 可关闭的状态 | 积分处理 |
|--------|--------------|----------|
| 普通用户 | 已下单 | 积分退还 |
| 店铺用户 | 已下单、已确认、已发货 | 积分退还 |

**关闭订单必须填写理由**（如：用户取消、商家缺货、其他）

### 6.3 订单流程

```
普通用户 → 浏览商品（卡片/表格）→ 选择商品 → 提交订单（填写收货信息）→ 积分扣除 → 订单状态：已下单
    ↓
店铺用户 → 确认订单 → 订单状态：已确认
    ↓
店铺用户 → 发货（填写快递类型+快递单号）→ 订单状态：已发货
    ↓
订单状态：已完成（店铺用户自行确认完成）

中途关闭：
- 已下单：普通用户/店铺用户可关闭（填写理由）→ 积分退还 → 已关闭
- 已确认：店铺用户可关闭（填写理由）→ 积分退还 → 已关闭
- 已发货：店铺用户可关闭（填写理由）→ 积分退还 → 已关闭
```

## 八、数据隔离

### 8.1 隔离级别

**表级别隔离（同一库，通过 shop_id 区分）**

### 8.2 隔离范围

| 数据类型 | 隔离范围 | 说明 |
|----------|----------|------|
| 商品分类 | 店铺级别 | 店铺用户管理本店分类 |
| 商品 | 店铺级别 | 店铺用户管理本店商品 |
| 用户 | 店铺级别 | 普通用户归属特定店铺 |
| 订单 | 店铺级别 | 订单与店铺绑定 |
| 积分 | 用户级别 | 每个用户的积分余额 |

### 8.3 隔离效果

- A 店铺的商品、用户、订单、积分，与 B 店铺**完全互不可见**
- 店铺用户只能操作本店数据
- 普通用户只能访问本店商品和查看个人订单

### 8.4 逻辑删除机制

| 场景 | 处理方式 |
|------|----------|
| 店铺用户账号删除 | **逻辑删除**（禁用状态），已登录的自动清理下线 |
| 普通用户账号删除 | **逻辑删除**（冻结状态），已登录的自动清理下线 |

## 九、UI 展现形式

### 9.1 液态玻璃效果（全局统一风格）

系统所有页面统一采用 **液态玻璃（Glassmorphism）** 设计风格。

#### 9.1.1 效果定义

| 效果维度 | 说明 | CSS 实现参考 |
|----------|------|--------------|
| 毛玻璃 | 半透明模糊背景 | `backdrop-filter: blur(12px)` |
| 透明度 | 背景 15%-25% 透明度 | `background: rgba(255,255,255,0.15)` |
| 边框光泽 | 1px 高光边框模拟玻璃边缘 | `border: 1px solid rgba(255,255,255,0.3)` |
| 阴影 | 柔和投影增加层次感 | `box-shadow: 0 8px 32px rgba(0,0,0,0.1)` |

#### 9.1.2 适用范围

- **页面背景**：所有页面统一液态玻璃背景
- **卡片组件**：商品卡片、订单卡片、统计卡片
- **弹窗/抽屉**：模态框、侧边抽屉
- **输入组件**：输入框、下拉选择、开关按钮
- **导航栏**：顶部导航、侧边菜单

### 9.2 颜色主题切换系统

#### 9.2.1 颜色选项（6种）

| 序号 | 主题名称 | 浅色模式 | 暗色模式 | 适用场景 |
|------|----------|----------|----------|----------|
| 1 | 科技蓝 | 冰川蓝 `#3B82F6` | 深空灰 `#1E293B` | 科技、企业 |
| 2 | 清新绿 | 薄荷绿 `#10B981` | 墨绿 `#064E3B` | 自然、健康 |
| 3 | 浪漫紫 | 薰衣草 `#8B5CF6` | 暗夜紫 `#4C1D95` | 时尚、艺术 |
| 4 | 活力粉 | 玫瑰粉 `#EC4899` | 莓果红 `#831843` | 电商、女性 |
| 5 | 纯净白 | 纯白 `#FFFFFF` | 纯黑 `#000000` | 通用 |
| 6 | 经典黑 | 纯白 `#FFFFFF` | 纯黑 `#000000` | 通用 |

#### 9.2.2 切换维度

| 维度 | 说明 |
|------|------|
| 明暗模式 | 浅色模式 / 暗色模式 |
| 玻璃颜色 | 液态玻璃背景的颜色基调 |
| 主题色 | 按钮、链接、强调元素的颜色 |

#### 9.2.3 切换交互

- **触发方式**：全局悬浮按钮（页面右下角）
- **交互流程**：点击悬浮按钮 → 弹出颜色选择面板 → 点击选择 → 实时预览 → 保存
- **记住偏好**：用户选择后保存到本地存储（localStorage），下次访问自动应用
- **扩展性**：颜色配置文件独立管理，方便后续新增颜色主题

### 9.3 卡片形式

- 商品列表：网格卡片展示（图片+名称+积分价格+库存）
- 订单列表：卡片列表展示（订单概要+状态+时间）

### 9.4 表格形式

- 商品列表：数据表格（列：图片、名称、分类、积分价格、库存、状态、操作）
- 订单列表：数据表格（列：订单号、商品、用户、积分、状态、下单时间、操作）

**切换机制**：页面提供「卡片/表格」切换按钮，用户可自由选择展示形式，系统记住用户偏好。

## 十、约束与限制

| 约束 | 说明 |
|------|------|
| 账号归属 | 每个普通用户只能属于一个店铺，创建后不可转移 |
| 积分货币 | 商品仅支持积分兑换，不支持现金支付 |
| 数据隔离 | 店铺间数据完全隔离，不可跨店查询 |
| 审核机制 | 店铺用户注册（审核账号）、店铺申请（审核店铺）、普通用户创建（审核用户）均需审核 |
| 店铺独立性 | 每个店铺独立管理自己的商品、用户、订单、积分 |
| 管理员权限 | 管理员不管理积分，仅能管理店铺和用户账号 |
| 库存管理 | 商品支持库存设置，**下单时扣减**，库存不足不允许下单 |
| 兑换次数限制 | 商品可设置「单用户最大兑换个数」，**默认0=无限制** |
| 积分操作记录 | 店铺用户增加/扣除普通用户积分时，**必须填写操作理由** |
| 订单超时提醒 | 订单「已下单」状态**超过7天**未确认，系统发送提醒 |
| 售后机制 | 人工处理（出现质量问题由店铺用户与普通用户线下协商） |
| 物流要求 | 实物商品需要买家填写收货信息，卖家填写快递信息 |
| 店铺账号与店铺关系 | 店铺用户账号与店铺是独立概念，账号通过审核后需单独申请店铺 |
| 店铺申请规则 | 申请被拒绝后可重新填写理由再次申请；店铺被冻结后需重新申请 |
| 店铺状态影响 | 店铺「申请中」或「已冻结」时，店铺用户不能操作上架商品、用户 |
