## 一、概述

### 1.1 设计思路

本数据模型服务于多店铺积分商城系统的UI原型设计，采用**表级别租户隔离**架构（通过 shop_id 区分）。核心实体包括：用户账号体系（管理员/店铺用户/普通用户）、店铺管理、商品分类与商品管理、订单兑换流程、积分变动追踪。

### 1.2 实体关系总览

```
用户账号 (user_account)
    ├── 管理员 (role=ADMIN) ───── 平台层，全局数据
    ├── 店铺用户 (role=SHOP_USER) ─┬─ 店铺层，拥有独立店铺
    │    └── 关联店铺 (shop) ─────┤
    └── 普通用户 (role=USER) ─────┴─ 用户层，归属特定店铺
              └── 关联店铺 (shop_user_map)

店铺 (shop) 1:N 商品 (goods)
店铺 (shop) 1:N 分类 (category)
店铺 (shop) 1:N 普通用户 (user_account)
商品 (goods) 1:N 订单项 (order_item)
订单 (order) 1:N 订单项 (order_item)
用户 (user_account) 1:N 积分记录 (point_log)
用户 (user_account) 1:N 消息 (message)
文件 (file) ── 被各业务表引用（商品图片、店铺Logo等）
```

### 1.3 逻辑删除说明

所有表均包含 `deleted` 字段，**标记删除时不物理删除数据**，仅将 `deleted` 设为 `DELETED`（已删除）状态，保留原数据供追溯。
